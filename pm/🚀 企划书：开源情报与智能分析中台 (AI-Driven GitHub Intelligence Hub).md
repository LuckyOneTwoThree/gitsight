# 🚀 企划书：开源情报与智能分析中台 (AI-Driven GitHub Intelligence Hub)

## 零、 核心价值与产品定位 (Positioning)

- **一句话定位**：**ProductHunt 的发现感 + 36Kr 的深度分析 + Notion 的文档产出**——专注全球开源项目的 AI 情报与分析中台（桌面端）。
- **产品形态**：**开源桌面端 AI 情报分析工具**——本地安装、数据私有、用户自带 LLM API Key，无需注册账号即可使用。
- **产品愿景**：消除开源世界的"信息差"与"理解门槛"，将非结构化的代码仓库转化为结构化的商业与技术洞察。
- **目标用户画像**：

  - **产品经理 (PM) / 创业者**：寻找竞品、逆向推导 PRD、挖掘未被满足的用户痛点。
  - **技术 Leader / 架构师**：高效技术选型、核心架构参考、学习前沿实现方案。
  - **独立开发者 (Indie Hacker)** ：寻找 Side Project 灵感、快速上手高门槛项目 (CodeWiki)。
  - **早期投资人 / 行业分析师 (VC/PE)** ：量化技术赛道热度，捕捉早期爆发的黑马项目。

- **核心差异化**：
  - 🖥️ **桌面端优先**：数据全部存储在本地 SQLite，无需注册、无需登录，隐私零风险。
  - 🤖 **15+ LLM 提供商支持**：OpenAI / Anthropic / Google / DeepSeek / 通义千问 / 智谱 / Kimi / 豆包 / 零一万物 / 阶跃星辰 / MiniMax / MiMo / 百川 / SiliconFlow / OpenRouter 及自定义接入——用户自带 API Key，按需选择，成本自控。
  - 🔓 **开源 MIT 协议**：代码完全开放，社区驱动迭代，可自行编译部署。

## 一、 产品功能架构 (Core Modules)

整个系统划分为 **发现 (Discovery) ➔ 深度解析 (Deep Dive) ➔ 对比决策 (Comparison) ➔ 全局洞察 (Landscape & Alerts)** 四大核心模块，**V1.0 已全部实现**。

### 模块 1：项目雷达与大盘 (Discovery) ✅ 已实现

彻底改变传统的按 Star 数排行的模式，引入**动态指标与情报评分系统**。

|功能模块|实现状态|核心机制|
| :---------| :------| :-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|**GitHub Trending 同步**|✅ 已实现|自动同步 GitHub Trending 每日/每周/每月榜单，支持按语言筛选。数据每 4 小时自动刷新，支持按需手动触发同步。|
|**Intel Score 情报评分**|✅ 已实现|自研多维度评分体系：**Velocity（增速维度 40%）+ Community（社区维度 35%）+ Maturity（成熟度维度 25%）**，输出 0-100 分值及 A+/A/B/C/D 等级，一眼识别黑马与优质项目。|
|**搜索与直达**|✅ 已实现|支持 GitHub URL 直接跳转（`github.com/owner/repo`、`git@github.com:owner/repo.git` 等格式自动解析），支持关键词搜索 GitHub 仓库，按 Star 排序返回结果。|
|**语义化智能搜索**|🔮 规划中|自然语言搜索，AI 意图匹配。当前基于 GitHub Search API 关键词搜索，语义搜索为未来迭代方向。|
|**AI 智能标签库**|🔮 规划中|AI 自动打上多维标签（如 `LLM-Agent`、`Rust`、`DevOps`、`B2B`），支持交叉筛选。当前使用 GitHub Topics 原生标签进行分类。|

### 模块 2：单点爆破与深度解析 (Deep Dive) ✅ 已实现

针对单一 Repo，根据不同角色的需求，**"一键生成千人千面的多维分析包"**。支持 **Fast（快速）/ Deep（深度）模式切换** 和 **中文/英文语言切换**。

|视角 / 产出物|Section Type|详情说明 (AI 自动生成)|目标群体|
| :--------------| :--------- | :------------------------------------------------------------------------------------| :----------------|
| **【TL;DR 摘要】**|`tldr`|30 秒看懂：项目是干嘛的？解决了什么痛点？核心优势是什么？|所有人|
| **【逆向 PRD】**|`reverse_prd`|从代码和文档反推需求文档、核心用户旅程、商业模式推断。|PM / 创业者|
| **【核心架构】**|`architecture`|核心架构梳理，**使用 Mermaid.js 实时渲染数据流图/架构图**，可视化展示系统设计。|开发者 / 架构师|
| **【CodeWiki】**|`code_wiki`|快速启动避坑指南、核心模块代码导读。|开发者 / 架构师|
| **【时光机分析】**|`timeline`|提取过去 1-2 年的核心 Release Notes，复盘该项目是如何从 MVP 演进到现在的。|PM / 架构师|
| **【技术栈清单】**|`tech_stack`|完整技术栈梳理，语言、框架、依赖关系分析。|开发者 / 架构师|
| **【社区健康度】**|`community`|License 风险分析、社区活跃度报告、贡献者生态评估。|投资人 / 创业者|
| **【贡献指南】**|`contribution_guide`|如何参与该项目贡献，Good First Issue 推荐，开发环境搭建指南。|开发者|
| **【供应链安全】**|`supply_chain`|依赖链安全分析，已知漏洞检测，许可证合规性审查。|安全工程师 / 架构师|

**关键特性**：
- **Fast / Deep 模式**：Fast 模式仅基于 README 和元数据快速生成；Deep 模式深入解析代码结构和更多上下文。
- **中英双语**：一键切换分析报告输出语言（`zh` / `en`）。
- **Mermaid.js 实时渲染**：架构分析章节生成的 Mermaid 代码在前端实时渲染为流程图，DOMPurify 安全过滤。
- **全量缓存 + 增量更新**：生成的报告自动缓存到本地 SQLite，仅当项目重大更新时标记为 stale 并提示重新生成。

### 模块 3：横向对比与机会挖掘 (Comparison) ✅ 已实现

将单一项目的理解升维到竞品分析，直接**输出商业价值**。

- **多项目对比**：支持同时对比 **最多 6 个项目**，自动去重。
- **4-Tab 对比矩阵**：
  - **Basic（基础指标）**：Stars / Forks / Issues / Watchers / License 等核心指标横向对比。
  - **Tech（技术栈）**：语言、框架、技术路线对比。
  - **Features（功能特性）**：基于 description / topics 推断的功能能力对比。
  - **Business（商业维度）**：商业化路径、进入壁垒、差异化空间对比。
- **Radar Chart 雷达图**：使用 Recharts 渲染多维度雷达图，直观展示项目间优劣势。
- **AI 深度对比报告**：调用 LLM 生成 12 章节结构化 Markdown 深度对比分析报告，包含一页纸决策摘要、市场定位对比、机会缺口分析、综合评分卡等。LLM 不可用时自动降级为基于元数据的确定性报告。
- **导出功能**：支持导出为 **Markdown** 和 **CSV** 格式。

### 模块 4：赛道全局视图与情报订阅 (Landscape & Alerts) ✅ 已实现

从"工具"向"情报中台"跨越的核心。

- **8 大赛道分类**：

  - 📚 **RAG / 知识检索**：RAG 框架、向量数据库、图数据库
  - 🎨 **前端 / UI**：UI 组件库、CSS 框架、全栈框架
  - ⚙️ **DevOps / 基础设施**：容器编排、可观测性、IaC / GitOps
  - 🗄️ **数据库 / 存储**：关系型、NoSQL、分析型引擎
  - 🛠️ **编程工具**：AI 编程助手、编辑器 / IDE、CLI / 终端
  - ⛓️ **Web3 / 区块链**：DeFi、链上基础设施、智能合约

## 二、 技术落地架构 (Technical Strategy)

### 技术栈全景

|层级|技术选型|说明|
| :---------| :-------------------------------| :-----------------------------------------------------------------------------------------------------------------|
|**桌面框架**|Electron 42|跨平台桌面应用，支持 Windows / macOS / Linux，系统托盘常驻。|
|**前端框架**|Next.js 16 + React 19 + TypeScript 5.7|Next.js 作为 Electron 内嵌渲染层，SSR/SSG 用于首屏加载优化。|
|**UI 组件**|Radix UI + Tailwind CSS 4|无障碍优先的组件库 + 原子化 CSS，shadcn/ui 风格。|
|**本地存储**|SQLite (better-sqlite3) + WAL 模式|单文件嵌入式数据库，WAL 模式支持并发读写，零运维成本。数据文件位于 `~/.repointel/repo-intel.db`。|
|**可视化**|Recharts + Mermaid.js + DOMPurify|Recharts 用于雷达图 / 趋势图；Mermaid.js 实时渲染架构流程图；DOMPurify 防止 XSS。|
|**多 LLM 接入**|15+ 提供商，用户自带 Key|OpenAI / Anthropic / Google / DeepSeek / 通义千问 / 智谱 / Kimi / 豆包 / 零一万物 / 阶跃星辰 / MiniMax / MiMo / 百川 / SiliconFlow / OpenRouter / 自定义。统一 OpenAI 兼容接口，无服务端代理。|
|**配置管理**|`~/.repointel/config.json`|JSON 文件存储所有配置（GitHub Token、LLM 提供商列表、API Key、语言、主题等），globalThis HMR 安全缓存。|
|**异步任务**|文件型 Job Store + 轮询进度|分析任务写入 SQLite job 表，前端轮询获取状态，无需 SSE / WebSocket / BullMQ。|
|**数据同步**|GitHub Trending HTML 解析 + GitHub Search API|定时同步 Trending 榜单 + 按 Topic 发现补充项目，4 小时自动刷新。|

### 核心架构决策

|决策点|选择|理由|
| :---------| :-------------------------------| :-----------------------------------------------------------------------------------------------------------------|
|**不使用 PostgreSQL / Redis**|SQLite + WAL|桌面端单用户场景，嵌入式数据库零运维，WAL 模式支持并发读，性能足够。|
|**不使用自建爬虫服务器**|直连 GitHub API + Trending HTML|桌面端无需中间层，用户自带 GitHub Token 提升限额，Trending 数据通过 HTML 解析获取。|
|**不使用 GitHub Archive**|实时 API + 本地缓存|桌面端不需要历史全量数据，实时 API + 本地 SQLite 缓存即可满足需求。|
|**不做用户认证**|单用户桌面，配置即身份|无需注册 / 登录 / 权限管理，config.json 即为用户身份，开箱即用。|
|**不做服务端 LLM 代理**|客户端直连 LLM API|用户自带 Key，桌面端直接调用 LLM API，无中间代理，零服务端成本，数据不经过第三方。|
|**不做 SSE / WebSocket**|轮询 + Job Store|桌面端单用户场景，轮询足够简单可靠，无需引入 SSE / WebSocket 的连接管理复杂度。|

### AI 上下文与 Token 成本控制

|核心挑战|解决策略|
| :---------| :-----------------------------------------------------------------------------------------------------------------|
|**GitHub 频率限制**|用户自带 GitHub Token（认证 5000 次/小时），本地 SQLite 缓存全量数据，核心业务只查本地。Trending 数据 4 小时同步一次。|
|**AI 上下文超限**|**分层降维打击**：第一层（Fast 模式）只解析 README、package.json、一级目录生成基础报告；第二层（Deep 模式）引入更多代码上下文。自动按模型上下文窗口截断输入，保留最关键内容。|
|**高昂的 Token 成本**|**全量缓存 + 增量更新**：一旦生成了某个 Repo 的分析报告，结果立刻缓存到本地 SQLite。后续查看直接调缓存，仅当项目重大更新时标记 stale 并提示重新生成。用户完全掌控成本——选择什么模型、生成什么章节，都由用户决定。|

### 总结结语

这个项目的核心壁垒不在于"用了什么大模型"，而在于 **"工程化编排能力"**（如何把代码、Issue、外部评价等散乱的数据，通过提示词工程组装成有价值的商业文档），**"数据缓存飞轮"**（先来的人生成了报告，本地缓存后持续受益），以及 **"开源社区生态"**（社区贡献模板、推送渠道、分析章节，形成正反馈）。

**桌面端优先的独特优势**：
- 🔒 **数据隐私**：所有数据存储在本地，不经任何第三方服务器，API Key 仅在本地使用。
- 💰 **零服务成本**：无需维护服务器、数据库、CDN，用户自带 Key，项目方零运营成本。
- ✈️ **离线能力**：已缓存的分析报告和项目数据可离线查看，不受网络限制。
- 🎯 **开箱即用**：无需注册、无需登录、无需配置服务器，下载安装即可使用。
